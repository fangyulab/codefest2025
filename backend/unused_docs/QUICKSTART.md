# 快速開始指南

## 🚀 立即開始

### 方法一：使用啟動腳本 (推薦)

```bash
cd /home/claude
./start.sh
```

### 方法二：手動啟動

```bash
cd /home/claude
python app.py
```

啟動後，後端會運行在 `http://127.0.0.1:8080`

## 📱 使用前端

在瀏覽器中開啟 `frontend.html` 檔案即可使用。

## 🧪 測試 API

如果想測試後端 API 是否正常運作：

```bash
# 確保後端正在運行
python test_api.py
```

## 📋 測試流程範例

1. **開啟兩個瀏覽器視窗**
   - 視窗 A: 開啟 `frontend.html`
   - 視窗 B: 開啟 `frontend.html` (使用無痕模式或其他瀏覽器)

2. **登入不同帳號**
   - 視窗 A: 登入帳號 `user1`
   - 視窗 B: 登入帳號 `user2`

3. **在視窗 A 發布求助**
   - 填寫求助標題：「需要協助，有人跟蹤」
   - 填寫內容：「在捷運站附近被人跟蹤，請幫忙」
   - 位置：`25.0330,121.5654` (台北車站座標)
   - 緊急程度：選擇「緊急 (5分鐘內)」
   - 標籤：選擇「跟蹤」、「捷運」
   - 聯絡方式：`0912345678`
   - 點擊「發布求助」

4. **在視窗 B 查看並回應**
   - 切換到「列表」tab
   - 應該會看到 user1 剛剛發布的求助
   - 點擊貼文查看詳情
   - 點擊「我正在前往協助」按鈕

5. **在視窗 A 確認**
   - 切換回視窗 A
   - 在列表中點擊自己的貼文
   - 應該會看到「1 人正在前往協助」
   - 當問題解決後，點擊「標記為已解決」

## 🔧 常見問題

### Q: 前端無法連線到後端？
A: 確認後端是否正在運行，應該在 `http://127.0.0.1:8080`

### Q: 如何清空所有資料？
A: 刪除 `users.txt`, `posts.txt`, `responses.txt` 三個檔案，系統會自動重新建立

### Q: 如何查看資料庫內容？
A: 直接用文字編輯器開啟 `.txt` 檔案即可查看 JSON 格式的資料

### Q: 為什麼我看不到其他人的求助？
A: 確認篩選條件，可能被「五公里內」或「標籤」篩選掉了

## 📊 資料檔案說明

系統會自動建立三個資料檔案：

- `users.txt`: 儲存使用者資訊
- `posts.txt`: 儲存求助貼文
- `responses.txt`: 儲存回應記錄 (誰正在前往協助)

這些檔案都是 JSON 格式，可以直接用文字編輯器查看或編輯。
